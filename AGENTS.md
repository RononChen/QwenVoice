# AGENTS.md

This is the canonical repository operating guide for coding agents working in QwenVoice. It replaces the removed tool-specific guide; do not recreate that file or add new references to it. Session-level user, system, and developer instructions still take priority over this file.

## Project Snapshot

QwenVoice is the repository identity for the Apple-platform Vocello product line. The repo currently contains a native Swift/Xcode codebase for macOS and iPhone, with shared engine semantics in `QwenVoiceCore`, macOS generation isolated in a bundled XPC service, and iPhone generation isolated in an ExtensionFoundation engine extension.

Current checked-in milestone facts from `project.yml` and the model contract:

- Repo name: `QwenVoice`
- Forward app name: `Vocello`
- macOS product: `Vocello.app` inside `Vocello-macos26.dmg`
- Marketing version/build in current `main`: `2.0.0` / `16`
- Deployment targets: `macOS 26.0+` and `iOS 26.0+`
- Official hardware floor: Mac mini M1 with 8 GB RAM, and iPhone 15 Pro for the deferred iPhone track
- Public macOS beta release is `Vocello v2.0.0-beta.1` for macOS 26 testers
- Public stable release remains `QwenVoice v1.2.3` for users who need macOS 15 support or do not want beta software
- The withdrawn RC1 GitHub release/tag must not be restored, linked, or advertised; public Vocello beta downloads use `v2.0.0-beta.1`.

The active public milestone is macOS-first. Keep iPhone compile-safe, but do not treat iPhone TestFlight/App Store proof as blocking unless the maintainer explicitly changes the release track.

## Non-Negotiables

- Do not reintroduce a repo-owned Python backend, Python setup path, standalone CLI surface, or bundled Python runtime.
- Do not move heavy model load, prewarm, generation, or clone preparation back into a UI process.
- Do not add iPhone artifacts to GitHub Releases; iPhone distribution is App Store/TestFlight only.
- Do not restore macOS 15 compatibility in current `main`; macOS 15 belongs to the historical v1.2.3 line.
- Do not restore the removed desktop-studio shell, inspector layout, oversized hero chrome, marketing-page UI, full-window footer player, or decorative glass/card redesigns.
- Do not make Release builds depend on `QW_TEST_SUPPORT`.
- Do not hand-edit generated Xcode project structure when `project.yml` is the source of truth; edit `project.yml` and regenerate.
- Do not run overlapping heavy `xcodebuild`, `qa.sh`, release packaging, live model generation, or benchmark jobs on the 8 GB development machine.
- Do not treat SourceKit or editor index errors as authoritative; use `xcodebuild`, `scripts/qa.sh`, and `scripts/build_foundation_targets.sh`.

## Architecture Boundaries

macOS app shell:

- `Sources/QwenVoiceApp.swift` composes app-wide services, initializes engine selection, and owns the Settings scene.
- `Sources/ContentView.swift` owns the macOS `NavigationSplitView`, toolbar/search chrome, sidebar selection, and persisted generation drafts.
- `Sources/Views/Generate/` owns Custom Voice, Voice Design, and Voice Cloning screens.
- `Sources/Views/Settings/SettingsView.swift` is the unified Models, Playback, and Storage settings surface. Do not reintroduce separate `ModelsView.swift` or `PreferencesView.swift`.
- `Sources/Views/Components/AppTheme.swift` centralizes macOS styling primitives.
- `Sources/ViewModels/` owns feature coordinators for generation and saved voices.
- `Sources/Services/` owns app-layer paths, database, downloads, audio, batch generation, startup, warmup, app commands, test stubs, and UI test support.

Shared app support:

- `Sources/SharedSupport/Services/GenerationPersistence.swift` owns cross-platform generation persistence handoff.
- `Sources/SharedSupport/ViewModels/AudioPlayerViewModel.swift` owns playback, live preview, handoff, and live-preview diagnostics.

Shared engine core:

- `Sources/QwenVoiceCore/TTSEngine.swift` defines frontend-safe engine state and engine protocols.
- `Sources/QwenVoiceCore/SemanticTypes.swift` and `GenerationSemantics.swift` define shared request, result, event, mode, clone, and generation semantics.
- `Sources/QwenVoiceCore/ContractBackedModelRegistry.swift` loads the checked-in model contract.
- `Sources/QwenVoiceBackendCore/` owns the QwenVoice-controlled backend facade and provenance for the upstream `mlx-audio-swift v0.1.2` seed.
- `Sources/QwenVoiceCore/MLXTTSEngine.swift`, `NativeEngineRuntime.swift`, `MLXModelLoadCoordinator.swift`, `NativeStreamingSynthesisSession.swift`, `NativeCloneSupport.swift`, and `AudioPreparation.swift` own the active native generation path. Normal production generation is quality-first full-result audio; live preview/streaming is secondary diagnostic or future preview work and must not affect the final waveform.
- `Sources/QwenVoiceCore/ExtensionEngine*` owns iPhone extension IPC, transport, host replacement, invalidation, and teardown.
- `Sources/QwenVoiceCore/IOSMemorySnapshot.swift` and `NativeMemoryPolicyResolver.swift` own shared memory-pressure vocabulary and policy.

macOS engine isolation:

- `Sources/QwenVoiceNative/TTSEngineStore.swift` is the macOS UI-facing engine store.
- `Sources/QwenVoiceNative/XPCNativeEngineClient.swift` owns app-side XPC connection coordination.
- `Sources/QwenVoiceNative/GenerationChunkBroker.swift` brokers streaming chunks into app playback.
- `Sources/QwenVoiceEngineSupport/` contains transport models and trust-policy helpers used by app and service.
- `Sources/QwenVoiceEngineService/EngineServiceHost.swift` hosts `MLXTTSEngine` from `QwenVoiceCore` inside the bundled XPC service.

iPhone isolation:

- `Sources/iOS/QVoiceiOSApp.swift`, `IOSAppBootstrap.swift`, and `QVoiceiOSRootView.swift` own the iPhone shell and dependency bootstrap.
- `Sources/iOS/TTSEngineStore.swift` is the iPhone UI-facing engine store.
- `Sources/iOS/VocelloEngineExtensionPoint.swift` owns monitor-backed extension discovery and preferred identity selection.
- `Sources/iOSEngineExtension/VocelloEngineExtensionHost.swift` hosts the isolated iPhone engine process.
- `Sources/iOS/IOSModelDeliveryActor.swift` and `Sources/iOSSupport/Services/IOSModelDeliverySupport.swift` own resumable iPhone model delivery.
- `Sources/iOSSupport/Services/AppPaths.swift` owns the App Group file layout.

## Model Contract

`Sources/Resources/qwenvoice_contract.json` is the source of truth for model metadata.
Each model and variant must include an immutable 40-character Hugging Face commit revision; macOS downloads use that revision instead of a moving branch name.

Current logical modes:

- `custom` -> Custom Voice -> output subfolder `CustomVoice`
- `design` -> Voice Design -> output subfolder `VoiceDesign`
- `clone` -> Voice Cloning -> output subfolder `Clones`

Speakers:

- default speaker: `aiden`
- speaker picker labels come from contract metadata and annotate native-language guidance, for example `Aiden - English native` and `Vivian - Chinese native`
- English-native speakers: `aiden`, `ryan`
- Chinese-native speakers: `vivian`, `serena`

Variant policy:

- macOS exposes `Speed / 4-bit` and `Quality / 8-bit` for all three modes.
- iPhone exposes Speed-only 4-bit variants.
- 8 GB/floor Macs default to and recommend Speed; mid/high-memory Macs default to and recommend Quality.
- Variant-specific IDs such as `pro_custom_speed` and `pro_custom_quality` own install status, downloads, repair/delete actions, and progress.
- Legacy base IDs such as `pro_custom`, `pro_design`, and `pro_clone` remain compatibility aliases to the hardware-recommended variant.

Model-folder families:

- Custom Voice: `Qwen3-TTS-12Hz-1.7B-CustomVoice-4bit` and `Qwen3-TTS-12Hz-1.7B-CustomVoice-8bit`
- Voice Design: `Qwen3-TTS-12Hz-1.7B-VoiceDesign-4bit` and `Qwen3-TTS-12Hz-1.7B-VoiceDesign-8bit`
- Voice Cloning: `Qwen3-TTS-12Hz-1.7B-Base-4bit` and `Qwen3-TTS-12Hz-1.7B-Base-8bit`


## Swift And Concurrency Notes

- Match existing Swift style: 4-space indentation, `PascalCase` types, `camelCase` members, descriptive enum cases, local formatting over mass-formatting.
- Use XCTest naming: test files named for behavior, methods prefixed with `test`.
- Prefer manifest-backed data over duplicated constants.
- `Task.detached` does not inherit cancellation. Wrap awaited detached tasks in `withTaskCancellationHandler` when cancellation must propagate.
- `AsyncThrowingStream` iteration should call `try Task.checkCancellation()` when the producer runs independently.
- `NSLock.lock()` and `unlock()` are unavailable in async Swift 6 contexts. Use `OSAllocatedUnfairLock<State>` for lock-protected state.
- `Task` operation closures are `@Sendable`; non-Sendable class captures may need real isolation or explicit `@unchecked Sendable` only when the class is already designed for concurrent use.
- Avoid broad environment-object invalidation during generation, prewarm, or playback.

## UI And Product Discipline

Vocello should feel warm, premium, native, and quiet. The product is local-first text-to-speech, not a SaaS dashboard or a marketing site inside an app.

UI rules:

- Prefer standard SwiftUI navigation, lists, forms, sheets, menus, toolbars, and system materials.
- Keep feature work inside the existing app structure unless the maintainer explicitly asks for a broader layout decision.
- Keep generation screens responsive during backend activity.
- Keep `Sources/Views/Components/AppTheme.swift` as the macOS styling center.
- Keep iPhone styling in the iPhone shell primitives/theme layer.
- Preserve VoiceOver labels, accessibility identifiers, keyboard behavior, Reduce Motion, and Reduce Transparency support.
- Do not use color alone for state.

Product messaging:

- Public README may lead with Vocello 2.0.0 beta 1 for macOS 26 testers, but it must keep QwenVoice v1.2.3 as the stable fallback for macOS 15 or non-beta users.
- Vocello is the forward macOS 26 rebrand and current app name on `main`.
- Do not claim iPhone is publicly shipping until the release track changes.
- Voice cloning copy must keep consent and rights responsibility clear.
