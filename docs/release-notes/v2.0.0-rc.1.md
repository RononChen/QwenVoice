# QwenVoice 2.0 RC1 — Vocello for macOS 26

This is the first release candidate of QwenVoice 2.0 and the first **Vocello-branded** macOS release. It ships the dual-variant model catalog (Speed 4-bit / Quality 8-bit per generation mode), a unified Settings surface, and is built exclusively for **macOS 26**.

QwenVoice 2.0 RC1 is a pre-release. The currently recommended stable build remains [QwenVoice 1.2.3](https://github.com/PowerBeef/QwenVoice/releases/tag/v1.2.3) for users who need macOS 15 support. Install RC1 only if you're on macOS 26 and want to test the next major version.

## What's new since 1.2.3

### Dual-variant model catalog

Each of the three generation modes — Custom Voice, Voice Design, Voice Cloning — now ships in two variants:

- **Speed (4-bit)** — recommended on 8 GB Macs. Faster than realtime on long-form text on the hardware floor.
- **Quality (8-bit)** — recommended on 16 GB and higher Macs. Higher fidelity at a 20-30% wall-time cost on lower-memory machines.

The Settings → Models picker remembers your variant choice per mode and surfaces a green check on the hardware-recommended option, plus a warning marker on the variant flagged "Heavy for your Mac" on 8 GB hardware. Each variant downloads, installs, repairs, and deletes independently.

### Unified Settings surface

The old Models tab and the Cmd+, Preferences window have been merged into a single System Settings-style page covering Models, Playback (auto-play, smooth playback), and Storage. Both the in-app sidebar `Settings` row and the macOS Cmd+, scene host the same view.

The Manage button on installed models opens a real AppKit menu (Reveal in Finder, Delete Model) with native macOS chrome, and matches the bezel of the Get / Repair buttons exactly.

### Resumable model downloads

If a model download is cancelled mid-flight or interrupted by a network failure, restarting it picks up where it stopped instead of redownloading from zero. This makes the first-time install of the 6-variant catalog (~15 GB) much more forgiving on flaky connections.

### Batch generation UI consistency

Long-running batch generations no longer display contradictory progress text. The "Generating item N of M / X of M complete" status line now stays internally consistent throughout the batch — a regression that affected clone batch runs in earlier builds.

### Backend memory tuning for 8 GB Macs

The native engine has been tuned for the 8 GB hardware floor. Cold-launch overhead and memory pressure during generation are both reduced, particularly for the Quality (8-bit) variants which previously stressed swap aggressively on Mac mini M1.

### Other improvements

- Pacing guidance for input text added to `docs/qwen_tone.md` (Qwen3-TTS doesn't parse SSML; pauses come from punctuation and line breaks).
- Manifest-side download size estimates populated for every variant so the Settings tab's `Get N.NN GB` button shows the right size before any download starts.
- macOS sidebar treats Settings as the canonical Models entry point; the deep-link from a sidebar item highlights the right variant row when relevant.

## Platform changes

- **macOS 26.0** is the minimum supported version. macOS 15 is **retired** and remains supported only on the already-shipped v1.2.3.
- `Vocello.app` inside `Vocello-macos26.dmg` is the only public artifact for this release. The DMG is Developer ID-signed, notarized, and stapled.
- Hardware floor: Mac mini M1 / 8 GB RAM. The application itself runs on higher tiers; the floor is what we optimize and benchmark against.

## Deferred from this release

- **Vocello for iPhone** is in active development and stays compile-safe on `main`, but TestFlight or App Store distribution is not part of this release. iPhone returns to the public release track once the shared core is proven stable on macOS.
- **0.6B model variants** are not supported. Vocello stays on the curated 1.7B Speed/Quality pair per mode. Users wanting smaller checkpoints can run them via the upstream `mlx-audio` CLI directly.

## Performance evidence

A 36-sample exhaustive UI benchmark across all 6 model variants on Mac mini M1 (8 GB) ran cleanly with **0 underruns, 0 stalls, 0 decode failures, 0 engine wedges**:

| Mode | Variant | long-form RTF |
|---|---|---:|
| Voice Design | Speed (4-bit) | **0.84** |
| Custom Voice | Speed (4-bit) | **0.90** |
| Voice Cloning | Speed (4-bit) | 1.13 |
| Voice Design | Quality (8-bit) | 1.12 |
| Custom Voice | Quality (8-bit) | 1.18 |
| Voice Cloning | Quality (8-bit) | 1.43 |

(RTF = wall-clock time / audio duration; below 1.00 is faster than realtime.)

## Installing RC1

1. Download `Vocello-macos26.dmg` from this release.
2. Open the DMG and drag Vocello.app to Applications.
3. The app is signed with a Developer ID and notarized — Gatekeeper should accept it without warnings.
4. On first launch, you'll be prompted to download the model variants you want to use. Each is 2.3-3.1 GB; the first model takes a few minutes on a typical home connection.
