## 0. 使用说明

这是一份可以直接交给外部设计师或输入生成式设计工具的完整需求书。设计方应先理解品牌和产品，再开始画图，
不要把“书”“声波”“字母 S”三个元素机械拼接。

此前项目内部制作过一轮黑白草图，但仅用于暴露问题，不作为造型参考，也不要求设计方在其基础上修改。新方案应从
本文定义的品牌目标重新出发，形成具有独立识别度和可注册潜力的原创标志。

生成式工具输出只能作为概念探索。最终 Logo 必须由设计人员重建为可编辑矢量图，检查几何关系、光学校正、缩小表现、
相似标志和字体授权，并保留完整的创作与权利记录。

## 1. 项目概述

### 1.1 产品名称

| 项目 | 内容 |
| --- | --- |
| 全球核心品牌 | `Sonafolio` |
| 标准拼写 | 首字母大写：`Sonafolio` |
| 禁止拼写 | `SonaFolio`、`Sona Folio`、`SONAFOLIO` 作为日常标准写法 |
| 简体中文辅助名 | `声卷` |
| 繁体中文辅助名 | `聲卷` |
| 中文标准读音 | `shēng juàn`，“卷”取书卷、篇章之意 |
| 中文品牌口号 | `让每一页，自然成声。` |
| 英文品牌口号 | `Every page, naturally voiced.` |
| 产品证明句 | `一篇长稿，一个完整 WAV。` |

`Sonafolio` 由声音联想的 `Sona` 与书页、文稿、篇册联想的 `folio` 组合而成。名称要传达的不是“一个 AI
玩具”，而是“把完整文稿整理成连续、自然、可交付声音的专业本地工作室”。

### 1.2 产品是什么

Sonafolio 是一款运行在 Apple Silicon Mac 上的本地语音创作工具，核心能力包括：

1. 一次提交完整长稿；
2. 内部自动规划、分段、生成、检查与合并；
3. 最终交付一个完整 WAV，并及时清理过程文件；
4. 支持保持音调的语速调节；
5. 支持自定义声音、声音设计和经过授权确认的声音克隆；
6. 模型下载完成后，主要生成工作在用户本机完成。

首发商业价值不是“AI”“模型参数”或“声音克隆”本身，而是：

> 把一篇很长的文稿，稳定地变成一条自然、连续、可直接使用的声音成品。

### 1.3 产品不是什么

Logo 不应让人首先联想到以下产品：

- 在线会议、语音聊天或电话工具；
- 音乐播放器、唱歌软件或 DJ 工具；
- 医疗心电监测；
- 普通电子书阅读器；
- 云端聊天机器人；
- 短视频娱乐变声器；
- 只服务助眠场景的应用。

## 2. 目标用户与使用情境

### 2.1 主要用户

- 需要把 13,000–18,000 字长稿制作成旁白的内容创作者；
- 有声故事、有声书、播客和历史叙事内容制作者；
- 视频配音、课程旁白和纪录片解说制作者；
- 对语速、连续性、隐私和本地处理有明确要求的专业或半专业用户；
- 希望先免费尝试分段生成，再为完整 WAV 与变速能力付费的个人创作者。

### 2.2 用户看到 Logo 时应产生的第一感受

按重要性从高到低：

1. **安静而专业：** 可以放心把一篇正式长稿交给它；
2. **连续而流畅：** 不是零碎片段，而是完整叙事；
3. **书面内容正在转化为声音：** 页面与声音是一体的过程；
4. **精确可控：** 属于生产工具，而不是一次性的 AI 特效；
5. **现代但不冰冷：** 有技术能力，也保留阅读和讲述的温度。

### 2.3 品牌气质比例

以下比例用于帮助设计判断，不是数学规范：

| 气质维度 | 目标 |
| --- | ---: |
| 专业 / 娱乐 | 75 / 25 |
| 沉静 / 活跃 | 70 / 30 |
| 人文编辑感 / 工程科技感 | 60 / 40 |
| 温暖 / 冷峻 | 60 / 40 |
| 克制 / 装饰 | 85 / 15 |
| 长篇叙事 / 短促即时 | 80 / 20 |

关键词：**书页、篇章、连续、讲述、自然节奏、本地、精确、沉静、可信。**

反关键词：**炫技、赛博、机器人、霓虹、娱乐变声、医疗、直播、嘈杂、卡通。**

## 3. 核心视觉命题

### 3.1 一句话视觉任务

> 设计一个“书页正在自然转化为连续声音”的独立符号，并让它在 16 px 时仍能被记住。

### 3.2 “书页声波”的正确理解

“书页声波”不是在一本打开的书上放一条波形，也不是把书本图标和音频图标左右排列。真正需要的是一个融合图形：

- 同一条边缘既能被理解为翻页，也能被理解为声音节奏；
- 同一块负形既能表现书脊或纸张间隙，也能表现声音流动；
- 图形在没有文字解释时仍然成立；
- 图形具有自己的轮廓，而不是依靠内部装饰线说明用途。

### 3.3 识别顺序

理想的识别顺序为：

1. 第一眼：一个简洁、稳定、值得记住的抽象符号；
2. 第二眼：发现它与书页、纸张或篇章有关；
3. 第三眼：发现其中存在声音、节奏或连续讲述的含义；
4. 长期使用后：用户不依赖文字也能把该图形认作 Sonafolio。

不要求每个人第一眼都同时说出“书”和“声音”。独特轮廓和品牌记忆优先于把所有功能画全。

## 4. 三个允许探索的设计方向

以下是概念方向，不是必须照抄的构图。设计方可以提出更好的第四方向，但必须解释它如何回应品牌命题。

### 方向 A：连续书页 / Continuous Folio

使用一条连续纸带或一个连续平面完成整个标志。纸带出现一次或两次克制的翻折，形成页面向声音流动的感觉；可以隐约
形成 `S` 的运动趋势，但不能画成明显字母，也不能像丝带、闪电、插头或道路。

目标：最直接表达“长稿连续成声”。

风险：容易过度复杂，或变成装饰性丝带。

### 方向 B：书脊负形 / Resonant Spine

以页片、书脊或折页形成主体，通过中心负形表现一次平静的语音起伏。声音节奏应像自然语言的呼吸与停连，不像心电图、
股票曲线或尖锐脉冲。

目标：书面内容与声音在同一中心结构中相遇。

风险：容易落入“打开的书 + 波形”的常见图标套路。

### 方向 C：篇章节奏 / Folio Rhythm Glyph

用两到三块宽厚的页片、页边或段落形态构成一个紧凑抽象字形。页片的长度、弧度和间距共同形成声音节奏；图形应带有
编辑、排版和长篇内容的气质，但不能像数据库、服务器堆栈、汉堡菜单或字母 `E`。

目标：形成最有品牌独占性的现代符号。

风险：过于抽象后，书页和声音语义会同时丢失。

## 5. 造型要求

### 5.1 基础形式

- 第一阶段只使用纯黑与纯白；
- 优先使用一个完整外轮廓或极少量相互依赖的形状；
- 内部负形必须承担概念意义，不能只是装饰；
- 图形总体接近方形，便于进入 macOS App 图标容器；
- 主标志本身不内置圆角方形背景，平台图标容器与品牌符号分开设计；
- 允许轻微不对称，以产生翻页和讲述的自然感；
- 避免过度几何对称、机械镜像和明显的数学波形。

### 5.2 线条与细节

- 16 px 下主要结构不能断裂；
- 最细有效结构缩小后应至少保持约 2 个可见像素；
- 不依赖页内文字线、纸张纹理、阴影或高光解释书页；
- 不出现三个以上需要分别辨认的小零件；
- 不使用尖锐且高频的锯齿波；
- 不用渐变弥补结构问题；
- 黑白反转后仍应保持识别度。

### 5.3 字母与文字

- 图形可以隐约包含 `S` 的运动方向，但不能成为普通字母 S 图标；
- 不强制加入 `F`，也不要求制作 `SF` 字母组合；
- App 图标内部不放 `Sonafolio`、`声卷`、`AI` 或 `TTS`；
- 字标与图形标分别设计，最后再形成组合规范。

---

## 6. 明确禁止的元素与误读

以下任一情况出现，都应视为需要推翻或重大修改，而不是进入配色掩盖问题：

- 麦克风、耳机、扬声器、喇叭口；
- 播放三角、暂停按钮、音乐音符；
- 机器人头像、聊天气泡、神经网络节点；
- 明显的 ECG 心电图或尖锐心跳线；
- 在打开的书上直接横放一条波形；
- 与书本分离的均衡器柱状条；
- 数据库圆柱、服务器堆栈、汉堡菜单；
- 电源插头、手指、羽毛笔、火焰、翅膀；
- 现有 Vocello 的 `V` 形标志；
- 与 Spotify、Apple Podcasts、Audible、Notion、ChatGPT 等知名产品产生明显轮廓联想；
- 依靠圆角方框、渐变或 3D 材质才成立的图案；
- 过度强调“助眠”的月亮、星星、枕头；
- 任何未经授权的现成图标、字体轮廓或商标改造。

## 7. 第一阶段输出任务

### 7.1 概念数量

第一阶段请输出 **9 个真正不同的黑白图形**：

- 方向 A：3 个；
- 方向 B：3 个；
- 方向 C：3 个。

“不同”指结构逻辑不同，而不是只改变圆角、倾斜角度或线条粗细。每个方案只展示一个标志，不要让生成工具在同一个小格
里混出多个不可分离的变体。

### 7.2 展示方式

每个方案需同时展示：

1. 黑图白底主视图；
2. 白图黑底反转图；
3. 128 px；
4. 64 px；
5. 32 px；
6. 16 px；
7. 轻度模糊或眯眼测试；
8. 放入无装饰的 macOS 图标方形容器后的占比测试。

第一阶段不要展示电脑模型、名片、玻璃墙、霓虹灯牌或 3D 压印效果。Mockup 会干扰对真实轮廓的判断。

### 7.3 每个方案的说明

每个方案附 3–5 句说明：

- 这个图形最先被看成什么；
- 书页和声音如何共用同一结构；
- 为什么适合长文本而不是即时聊天；
- 16 px 时保留什么、牺牲什么；
- 它与常见“书 + 波形”图标的区别。

## 8. 评审标准

所有方案按 100 分评审：

| 项目 | 权重 | 判断问题 |
| --- | ---: | --- |
| 独特性与品牌记忆 | 25 | 看过一次后，能否从一组语音应用中再次找出？ |
| 书页与声音的融合 | 20 | 两个概念是否共用结构，而非机械拼接？ |
| 16 px 缩小表现 | 20 | Dock、小尺寸列表和菜单中是否仍有稳定轮廓？ |
| 品牌气质 | 15 | 是否沉静、专业、连续、有人文编辑感？ |
| 国际化与中性 | 10 | 是否不依赖中文文化典故，也不会产生明显负面误读？ |
| 扩展性 | 10 | 是否适合 App 图标、网站、文档、水印和动效？ |

进入第二阶段的建议门槛为 80 分，并且不能触发第 6 节中的致命误读。高分但明显像现有商标的方案仍然淘汰。

### 8.1 一秒识别测试

向未参与项目的人展示 1 秒后隐藏，询问：

- 你记住了什么形状？
- 它更像阅读、声音、音乐、医疗还是数据工具？
- 你能否从另外 8 个轮廓中重新找到它？

不要求受试者准确说出产品功能，但不应集中误判为医疗、数据库、音乐播放器或聊天工具。

### 8.2 缩小测试

在 128、64、32、16 px 检查：

- 外轮廓是否仍完整；
- 负形是否堵塞；
- 是否变成普通字母；
- 是否丢失翻页方向；
- 黑白反转时是否出现视觉重量失衡。

## 9. 第二阶段：入选方案精修

第一阶段评出 3 个方案后，每个方案最多进行两轮结构修改。此时仍保持黑白，不进入色彩和材质。

精修内容包括：

- 统一曲率和转折逻辑；
- 修正左右与上下视觉重量；
- 调整负形开口，防止小尺寸堵塞；
- 去除生成式图像常见的不规则曲线和伪对称；
- 建立可解释的网格，但不为了网格牺牲自然感；
- 同时制作几何版和光学校正版，最终以视觉效果为准；
- 做一次公开图形近似检索，排除明显冲突。

第二阶段最终只保留一个主方向。没有选定轮廓前，不讨论深色玻璃、纸张材质、立体阴影或 Liquid Glass。

## 10. 第三阶段：品牌系统与 App 图标

轮廓确定后，再开展以下工作。

### 10.1 颜色方向

首轮候选色仅用于探索，不是强制最终值：

| 用途 | 候选色 | 含义 |
| --- | --- | --- |
| 深色主色 | `#17172A` | 沉静、长篇阅读、专业工具 |
| 暖纸白 | `#F2E7D5` | 纸张、人文、自然讲述 |
| 少量强调 | `#D7A65A` | 温度、完成、声音能量 |

要求：

- 主标志必须先在单色下成立；
- 常规品牌版尽量不超过两种主色和一种小面积强调色；
- 颜色不应让产品变成睡眠专用、复古出版社或奢侈品；
- 深色和浅色背景都需提供标准版本；
- 渐变只能服务 App 图标空间层次，不能用于修复轮廓。

### 10.2 字标与组合

需要设计或明确以下组合：

1. 单独图形标；
2. 图形标 + `Sonafolio` 横排英文；
3. `Sonafolio 声卷` 简体中文组合；
4. `Sonafolio 聲卷` 繁体中文组合；
5. 仅英文正式字标；
6. 深底、浅底与单色版本。

英文字标应具备现代编辑感和良好长文本产品气质，避免科幻字体、圆润儿童字体和常见 AI 科技字。中文辅助名不应在视觉上
压过 `Sonafolio`。所有字体必须具备明确的商业授权；若修改字体轮廓，需保留原字体和修改记录。

### 10.3 App 图标

品牌符号与 App 图标是两个层级：

- 主 Logo 是可独立使用的矢量符号；
- App 图标是在平台容器、背景色、留白和层次中应用该符号；
- 不把平台圆角直接画进主 Logo；
- 不依靠很细的阴影、纸纹或透明层保持识别；
- 最终按当时 Xcode Asset Catalog 和 App Store 的正式要求导出，不在概念阶段锁死平台细节。

## 15. 项目方最终决策清单

在确认 Logo 前，项目方逐项签字或记录结论：
